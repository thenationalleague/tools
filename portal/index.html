# RTDB Migration Note: clubRole → jobTitle **Date:** 08/04/2026 **Status:** RTDB already updated. Frontend pages still need patching. ## What changed The `clubRole` field has been removed from all user records in the RTDB. The `jobTitle` field is now the single source of truth for a user's title/role, regardless of whether they are NL staff, admin, or club. Previously both fields existed on most users with identical values. The exception was the superadmin account where `clubRole` was empty but `jobTitle` had the correct value. The duplication was confusing and `clubRole` is a misleading name for NL staff who aren't at a club. ## RTDB structure (before → after) ``` BEFORE: users/{uid}/clubRole: "Chief Operating Officer" users/{uid}/jobTitle: "Chief Operating Officer" AFTER: users/{uid}/jobTitle: "Chief Operating Officer" (clubRole removed entirely) ``` ## What needs patching Search each file for any reference to `clubRole` and replace with `jobTitle`. This includes: - Reading from Firebase: `snap.val().clubRole` → `snap.val().jobTitle` - Writing to Firebase: any user creation/update that sets `clubRole` → set `jobTitle` instead - Display labels: if any UI says "Club Role" as a label, change to "Job Title" or just "Title" - Form fields: any input with id/name `clubRole` → `jobTitle` ## Pages to patch 1. **Portal** (`/tools/portal/index.html`) - Admin panel user table — likely shows title in a column or detail view - Invite flow — if it sets clubRole on new user creation - User profile/edit — if users can edit their own title - Any audit log display that references clubRole 2. **Holiday & Lieu** (`/tools/holiday-lieu/`) - Request display — if it shows the requester's title - Admin view — if it shows staff titles alongside names - Any user card or profile summary 3. **Auth guard** (`/tools/auth-guard.js`) - Check if it reads clubRole for topbar display or state 4. **Vacancies** (`/tools/vacancies/`) - If it shows the submitter's title anywhere ## How to patch In each file, do a find-and-replace: - `clubRole` → `jobTitle` (in JS variable names, Firebase paths, object keys) - `"Club Role"` or `"Club role"` → `"Job Title"` or `"Title"` (in UI labels) - Check form submission payloads that write to Firebase — ensure they write `jobTitle` not `clubRole` ## Also done in this migration (for context) - All user tool access strings normalised from flat format (`"access"`, `"admin"`, `"hidden"`) to object format (`{ access: true, admin: false }`) — 60 fields across 5 users - Empty `judgements` node added to RTDB for the new Judgements & Decisions tool - RTDB rules updated with `"judgements": { ".read": true, ".write": "auth != null" }`
